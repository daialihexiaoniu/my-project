## Git 基本操作
### Git 和 Github 的区别 * Git 是一个免费、开源的分布式版本控制系统 * Github 是用 Git 做版本控制的代码托管平台
### Git 和 SVN 的区别 * Git 和 SVN 都是版本控制系统，两者最主要的区别是：Git 是分布式版本控制系统，而 SVN 是集中式版本控制系统。 * Git 把内容按照元数据存储，而 SVN 按文件方式存储。 * Git 的分支和 SVN 的分支不同：Git 的分支相对比较简单，很容易发现未合并的分支，而 SVN 需要手动运行代码来确认代码是否被合并。
### Git 的一些常用命令操作 ```git clone <版本库的网址> ```从 Git 仓库中克隆一个项目。 ![git clone](./img/git-01.png) ![git clone](./img/git-02.png) ```git clone <版本库的网址> <本地目录名>``` ![git clone](./img/git-03.png)
### Git 的一些常用命令操作 ``` git config --global user.name xxxx git config --global user.email xxxx ```
### Git 的一些常用命令操作 ```git init``` 在目录中初始化新仓库，初始化后会在目录中出现一个 .git 的目录，所有Git需要的数据和资源都存放在这个目录中。 ![git init](./img/git-04.png) ![git init](./img/git-05.png) ![git init](./img/git-06.png)
### Git 的一些常用命令操作 ```git status``` 查看文件处于什么状态。 ![git init](./img/git-07.png) ![git init](./img/git-08.png)
### Git 的一些常用命令操作 ```git add [path] ``` 将要提交的文件的信息添加到索引库中。 ![git init](./img/git-10.png) ```git add -u [path]``` 把[path]中所有跟踪文件中被修改过或已删除文件的信息添加到索引库。它不会处理那些不被跟踪的文件。
### Git 的一些常用命令操作 ```git commit```将索引的当前内容与描述更改的用户和日志消息一起存储在新的提交中。 ```git commit -m "[commit message]"``` ![git init](./img/git-11.png)
### Git 的一些常用命令操作 ```git rm```命令用于从工作区和索引中删除文件。 ![git init](./img/git-14.png)
### Git 的一些常用命令操作 ```git diff```命令用于显示提交和工作树等之间的更改。此命令比较的是工作目录中当前文件和暂存区域快照之间的差异,也就是修改之后还没有暂存起来的变化内容。 ![git init](./img/git-12.png)
### Git 的一些常用命令操作 ```git branch```命令用于列出，创建或删除分支。 ![git init](./img/git-15.png)
### Git 的一些常用命令操作 ```git pull```用于拉取远端仓库的代码。 ```git push```用于将本地分支的更新，推送到远程主机。
### Git 的一些常用命令操作 ```git reset```用于将当前HEAD复位到指定状态。一般用于撤消之前的一些操作 ```git log```用于显示提交日志信息。 ```git stash```命令用于将更改储藏在脏工作目录中。(读取 git stash pop)
